# Deep Learning Docs

Documentation repo for personal deep learning study projects. Maintained by
Dmitry Grigoryev.

Read through both Obsidian locally and the GitHub web view — formatting choices
that break either surface are unacceptable.

## Structure

- [projects/](projects/README.md) — one folder per project, named to match the code repo; each has a `README.md`.
- `ideas/` — experiment ideas and directions not yet tied to a specific project.
- `papers/` — paper notes and reading list.

## Conventions

- Standard markdown links (`[label](file.md)`) — no Obsidian wikilinks.
- Terse bullet points over prose paragraphs.
- Lead with the non-obvious finding or sharp edge; skip introductory framing.
- No binary files — screenshots and diagrams stay on disk untracked or link externally.
